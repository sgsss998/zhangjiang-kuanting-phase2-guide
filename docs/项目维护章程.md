# 项目维护章程

> 本章程用于固定项目边界、维护权限和隐私红线。  
> 适用范围仅限于“张江城投宽庭二期入住生活指南”项目。

## 1. 项目定位

这不是物业公告，也不是商业宣传页，而是一份由住户自发维护的民间生活指南。

它要解决的问题是：一个新住户刚搬进张江城投宽庭二期后，怎样最快知道吃饭、快递、外卖、垃圾、停车、充电、健身、物业、维修、周边生活配套和邻里互助的信息。

## 2. 项目负责人职责

维护者作为项目负责人，负责：

- 主动规划版本路线，而不是等待零散派活。
- 主动判断哪些信息需要收集、核实、归档和发布。
- 主动向住户或资料提供者提出具体收集任务。
- 对所有公开内容做隐私检查。
- 将公开安全内容更新到 GitHub repo。
- 同步维护适合微信群传播的精美版 PDF/Word。
- 将不确定信息放入待确认清单，不包装成事实。

## 3. 授权范围

在本项目范围内，维护者可以直接：

- 修改公开版 Markdown 文档。
- 新增公开安全的图片、菜单、点位说明和住户体验。
- 更新待确认清单、版本路线图、资料来源。
- commit 并 push 到 `sgsss998/zhangjiang-kuanting-phase2-guide`。
- 为微信群制作 PDF/Word 版本。

不需要逐次确认的前提是：内容必须公开安全，且必须与张江城投宽庭二期入住生活指南直接相关。

## 4. 活动边界

维护权限仅限于本项目。

不得因为本项目授权而改动：

- 用户其他 GitHub repo。
- 用户其他工作项目。
- 用户其他个人资料库。
- 与张江城投宽庭二期生活指南无关的文件。

## 5. 隐私红线

以下内容不得进入公开 repo、微信群 PDF/Word、公网上传图片或公开文档：

- 姓名、身份证号、手机号、微信号、邮箱。
- 具体房号、住户身份、门牌级个人信息。
- 租赁合同、备案通知、税务清单、证照确认截图。
- 租金、面积、合同周期、合同编号、备案编号。
- 订单号、付款码、个人二维码、快递单号。
- 能反推出某个具体住户身份或住址的信息。

如原始材料含上述信息，只能本地私有归档，不得公开；公开版本只保留社区级、商户级、公共配套级信息。

## 6. 信息写作原则

1. 新住户第一：优先回答“我现在该去哪里、找谁、怎么做”。
2. 接地气：少写宏大介绍，多写具体点位、价格、路径、注意事项。
3. 来源分级：官方信息、住户实测、待核实必须分开。
4. 可维护：每个板块都要留下待补充项，方便持续更新。
5. 不制造权威感：民间版指南不替代物业、商户或官方通知。

## 7. 交付形态

长期维护三种形态：

- GitHub Markdown：主版本，便于持续更新和公开协作。
- 微信群 PDF：适合群内转发、手机阅读和新住户快速浏览。
- Word 版本：适合后续人工编辑、打印、整理成正式共享文档。

PDF/Word 的内容以公开 Markdown 为基础，但允许更精美的目录、封面、排版、表格和分栏。

## 8. 用户协作方式

用户不需要整理成正式文档，只需要持续提供原始资料：

- 图片
- 菜单
- 地址
- 商户名
- 群友反馈
- 现场公示
- 物业通知
- 自己的实测体验

维护者负责归档、脱敏、判断可信度、改写、排版、发布和版本管理。
